Popup v2 — the launcher redesign

The popup drops its desktop-utility chrome (sync bar, folder sidebar, hint row, twin footer buttons, version strip) and becomes a compact single-column launcher: search-first, keyboard-driven, folder chips, and an inline per-language detail — the same visual vocabulary the mobile companion already ships. Static mock, exact token values; no product code written yet.

480×600 window (was 600×~560) rows 60px touch targets (was ~40px) 0-click copy: type → Enter body copy per language — new, parity with dashboard tokens: azure v1.1, light only

03Cold open — skeleton instead of a lie

  • Today the popup flashes "No snippets found" on every open while Supabase loads. v2 hydrates instantly from the local cache and reconciles in the background.
  • Skeleton rows appear only on a cold cache (first run / after sign-in); reduced-motion gets static placeholders.
  • Status chip doubles as the sync indicator: azure pulse while syncing, green when done, red "Sync failed — tap to retry".

Deliberately unchanged

  • Row click still copies the shortcut and bumps usage stats — same semantics on both surfaces.
  • Read-only stays read-only: no CRUD returns to the popup; management hands off to the dashboard.
  • content.js expansion caches, auth gate (SSO + OTP), changelog modal: functionally untouched, restyled only.
  • Trigger prefix always rendered from your settings — the mock's :: is dynamic, never hardcoded (fixes the tab glyphs that are hardcoded today).

Token diff (registered in DESIGN_SYSTEM.md first)

  • lang-fr#0D9488 / #F0FDFA — FR becomes first-class teal in the extension sheet (docs already say so; popup + Sprintbrain.html align)
  • toast-bgrgba(24,24,27,.92) — tokenizes the existing toast literal
  • scrimrgba(0,0,0,.5) — modal backdrop token
  • dur-loop1200ms — skeleton shimmer + syncing pulse
  • stagger20ms/row — list entrance, capped at 8 rows, off under reduced motion
Frames are pixel specs at 480×600 (Chrome popup limit is 800×600; height is already at the cap) with real library data. Approve and implementation starts on popup.html + popup.js — surgical, shared-core-safe, verified on both surfaces.
