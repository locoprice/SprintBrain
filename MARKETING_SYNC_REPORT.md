## 2. The feature, as actually built (source of truth for the copy)

Verified against the live components so the copy makes no claim the app can't back:

- **The folder is the permission boundary.** Snippets, formulas, and prompts inside a shared folder
  inherit its access ([folderShares.ts](app/src/lib/folderShares.ts), [FolderShareModal.tsx](app/src/features/org/FolderShareModal.tsx)).
- **Share with the whole team or specific teammates** — right-click a folder → "Share with team…"
  ([FolderContextMenu.tsx](app/src/features/snippets/FolderContextMenu.tsx)).
- **Three access levels: View / Edit / Owner.** "Can use" / "use and modify" / "full control incl.
  sharing". Whole-team grants are View/Edit only; individuals can be Owner.
- **Only the folder owner or a team admin** can change sharing.
- **At-a-glance badges** in the folder tree: globe = whole team, people icon = N teammates, private =
  no badge ([SnippetFolderTree.tsx](app/src/features/snippets/SnippetFolderTree.tsx)).
- **Enforced server-side by RLS**, on both dashboard and extension (the extension reads
  `accessible_snippets()`).
- Sharing is **managed in the dashboard**; shared folders surface automatically in every teammate's
  extension and web library. **Mobile has no sharing UI** — copy is careful not to imply it does.
- This replaced the old per-snippet "Share with team" toggle (`is_shared`, dropped in B6).
